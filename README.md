# 📌 Project Overview
This project focuses on analyzing smart city bike-sharing data using Power BI. The goal is to transform raw data into meaningful insights that help understand station performance, operational efficiency, and revenue trends across cities.
The project covers the complete data analytics workflow including:
Data cleaning
Data modeling
DAX calculations
Data visualization
Insight generation
# 📂 Data Source
The dataset used in this project is related to smart city bike-sharing systems, which includes data collected from multiple bike stations across different cities.
Source Type: Public/Provided dataset (CSV file)
Data Format: Structured data (tabular format)
Data Contents:
City information
Station details
Ride records
Distance traveled (KM)
Revenue generated
Date and time of usage
This dataset was imported into Power BI and used for analysis, transformation, and visualization to derive actionable insights on urban mobility and operational efficiency.

Documentation: Power BI Report / PDF Summary
# 🧹 Data Cleaning & Preparation
Data cleaning and preparation were performed using Power Query Editor in Power BI to ensure data accuracy, consistency, and usability for analysis.
🔧 Steps Performed:
Removed Duplicates
Identified and eliminated duplicate records to maintain data integrity.
Handled Missing Values
Replaced or removed null values where necessary
Ensured no critical fields were left incomplete
Data Type Conversion
Converted columns to appropriate data types (e.g., Date, Number, Text) for accurate analysis.
Column Renaming & Formatting
Renamed columns for better readability and standardized naming conventions.
Filtering & Cleaning Data
Removed irrelevant or inconsistent records to improve dataset quality.
Merging & Appending Tables
Combined multiple datasets where required to create a unified data model.
Created New Columns
Derived additional fields (e.g., revenue per km, usage categories) to support analysis.
# 🔍 Exploratory Data Analysis (EDA)
Exploratory Data Analysis (EDA) was performed to understand the dataset and identify key patterns, trends, and anomalies. It included analyzing revenue trends over time, comparing city and station performance, and detecting outliers. This step helped in selecting important metrics and guiding the creation of effective visualizations and insights.
# 💡 Key Insights
Top-performing cities generated significantly higher revenue per KM, indicating better operational efficiency.
A few cities consistently showed low performance, highlighting areas needing improvement.
Revenue trends revealed steady growth over time, with some seasonal fluctuations.
Certain stations had high usage demand, while others remained underutilized.
Variations in revenue per KM across cities indicate differences in pricing or usage efficiency
# 🚀 Recommendations
Improve operations in underperforming cities by optimizing station locations and increasing bike availability.
Redistribute bikes to high-demand stations to meet user needs and reduce shortages.
Implement dynamic pricing strategies to improve revenue in low-performing areas.
Promote usage through offers and awareness campaigns in low-demand regions.
Use data insights for predictive planning to handle seasonal demand and improve efficiency.
